# 目录与修改指南

这份文档是给后续维护这个单书站用的，目标只有两个：

1. 看懂 `book-kb-领导者的意识进化` 里每个目录/文件大概是干什么的。
2. 知道“我想改某一类东西，应该去哪里改”。

## 先记住 4 条

1. `generate_site.py` 是这套站点的数据生成中枢。
2. `vault/`、`web/public/vault/`、`web/src/data/graphData.js`、`web/public/chapter-images/` 都会被 `generate_site.py` 重新生成。
3. 所以长期正确的改法，不是直接改生成结果，而是改上游素材或改 `generate_site.py`，然后重新跑生成。
4. `web/src/components/*.vue` 才是前端页面和交互本体，改 UI、布局、交互逻辑主要在这里。

## 改什么去哪里改

- 改文章正文内容：优先改上游素材，再运行 `python .\\generate_site.py`。
- 临时补一篇卡片文案：可以直接改 `web/public/vault/...`；但下次跑生成会被覆盖，最好同步回上游。
- 改首页文案、推荐阅读路径、快捷入口、作者信息、统计数字：优先改 `generate_site.py` 里的 `site = {...}`，再重跑生成。
- 改知识图谱节点、连线、标签、目录结构、搜索数据：改 `generate_site.py`，再重跑生成。
- 改页面布局和交互：改 `web/src/App.vue` 或 `web/src/components/*.vue`。
- 改全站配色、字体、圆角、阴影等基础视觉：优先改 `web/src/App.vue` 里的 `:root` CSS 变量。
- 改首页 SEO、站点默认标题/描述：改 `web/index.html`。
- 改详情页静态预渲染、`sitemap.xml`、`robots.txt`：改 `web/scripts/prerender.js`。
- 改二维码图片：替换 `web/public/social/*.png`。
- 改章节配图顺序：改 `generate_site.py` 里的 `CHAPTER_IMAGE_ORDER` / `OVERVIEW_IMAGE_INDEX`，再重跑生成。
- 改 Netlify 构建和发布配置：改 `netlify.toml`。
- 改 SPA 回退规则：改 `web/public/_redirects`。

## 根目录说明

- `generate_site.py`
  这套站点最重要的脚本。负责把上游书籍素材、K 卡/N 卡、结构化知识和小红书配图，整理成站点实际使用的数据和 Markdown。
  会输出到：
  `vault/`
  `web/public/vault/`
  `web/public/chapter-images/`
  `web/src/data/graphData.js`

- `README.md`
  原有项目说明，主要是怎么生成、怎么本地预览、上游素材来自哪里。

- `netlify.toml`
  Netlify 部署配置。
  包括构建命令、发布目录，以及 HTML / 静态资源缓存策略。

- `目录与修改指南.md`
  你现在看的这份维护说明。

- `vault/`
  生成后的 Markdown 主副本，按主题/章节/概念/方法/场景/人物分目录存放。
  更像“本项目内的知识库镜像”。
  注意：它是生成产物，不建议把这里当长期唯一源头。

- `web/`
  Vue + Vite 前端站点。
  负责展示首页、知识图谱、文章阅读页、SEO 预渲染和静态资源。

- `__pycache__/`
  Python 运行 `generate_site.py` 后产生的缓存目录。
  不用手改。

## `web/` 目录说明

### `web/package.json`

- 前端项目脚本和依赖定义。
- 常用命令：
  `npm run dev`
  `npm run build`
  `npm run preview`

### `web/package-lock.json`

- npm 依赖锁文件。
- 不手写内容，安装依赖时自动更新。

### `web/vite.config.js`

- Vite 配置。
- 现在内容很轻，主要启用了 Vue 插件。

### `web/index.html`

- 站点 HTML 壳文件。
- 改站点默认 `<title>`、首页 meta description、OG 信息、favicon、默认主题色，在这里改。
- 注意：这里是“首页/SPA 壳”的默认 SEO，不是每篇文章页的 SEO。

### `web/.gitignore`

- 前端目录自己的忽略规则。
- `node_modules/`、`dist/`、日志等都在这里忽略。

### `web/scripts/`

- `prerender.js`
  构建后的预渲染脚本。
  在 `npm run build` 时运行，读取 `web/public/vault/` 的 Markdown 和 `graphData.js`，为每个文章生成静态详情页，并输出：
  `web/dist/note/.../index.html`
  `web/dist/sitemap.xml`
  `web/dist/robots.txt`
  如果要改详情页 SEO、canonical、JSON-LD、静态详情页模板，就改这里。

### `web/src/`

- `main.js`
  Vue 入口文件，负责挂载 `App.vue`。

- `App.vue`

  前端直接消费的数据文件。

  `TOC`
  `HOME_SECTIONS`

  `NODE_IMAGES`

## 常见修改场景

### 1. 我想改某一篇文章正文

- 最稳妥：改上游素材，再执行：
  `python .\\generate_site.py`
- 只想临时在站里热修：改 `web/public/vault/对应目录/对应文件.md`
- 如果你希望本目录里的镜像也一致，再同步改 `vault/对应目录/对应文件.md`

### 2. 我想改首页上的文案和入口

- 改 `generate_site.py`
- 重点看：
  `site = {...}`
  `home_sections = [...]`
  `toc = [...]`
- 改完重跑：
  `python .\\generate_site.py`

### 3. 我想改知识图谱关系

- 改 `generate_site.py`
- 重点看：
  `specs`
  `chapter_keywords`
  `add_edge(...)`
  以及生成 `edges` / `link_labels` 的逻辑
- 改完重跑：
  `python .\\generate_site.py`

### 4. 我想改首页、图谱页、文章页的样式

- 改 `web/src/App.vue`
- 改 `web/src/components/HomeView.vue`
- 改 `web/src/components/KnowledgeGraph.vue`
- 改 `web/src/components/ArticleReader.vue`
- 改完前端预览：
  `cd .\\web`
  `npm run dev`

### 5. 我想改 SEO / 静态详情页

- 改首页默认 SEO：`web/index.html`
- 改文章静态页 SEO：`web/scripts/prerender.js`
- 改完重新构建：
  `cd .\\web`
  `npm run build`

### 6. 我想改二维码、缩略图、分享图

- 改二维码：替换 `web/public/social/*.png`
- 改默认分享图：替换 `web/public/og-image.png`
- 改 favicon：替换 `web/public/favicon.svg`
- 改章节缩略图逻辑：改 `generate_site.py`

## 最后给维护时的建议

- 想长期保留的内容修改，优先改上游素材或改生成逻辑，不要只改生成结果。
- 想改“页面怎么长、怎么动、怎么排”，去 `web/src/components/`。
- 想改“站里有哪些卡片、卡片怎么连接、首页怎么组织”，去 `generate_site.py`。
- 想改“部署和 SEO 出来的 HTML 长什么样”，看 `web/index.html`、`web/scripts/prerender.js`、`netlify.toml`。
